登录概述

登录的业务流程

  1. 在登录页面输入用户名和密码
  2. 调用后台接口进行验证
  3. 通过验证之后，根据后台的响应状态跳转到项目主页

登录业务的相关技术点

  1. http是无状态的
    • 通过cookie在客户端记录状态（前端与后台不存在跨域问题）
    • 通过session在服务器端记录状态（前端与后台不存在跨域问题）
    • 通过token方式维持状态（前端与后台存在跨域问题）

登录 —token原理分析

  1. 客户端登陆页面输入用户名和密码进行登录
  2. 服务器通过验证之后生成该用户的token并返回
  3. 客户端存储该token
  4. 客户端之后所有的请求都携带该token发送请求
  5. 服务器端验证token是否通过

退出功能的实现

退出功能实现原理

基于token的方式实现退出比较简单，只需要销毁本地的token即可。这样后续的请求就不会携带token，必须重新登录生成新的token之后才可以访问页面。
