跨域

什么是跨域？

  • 违背了同源策略就是跨域
    • 同源策略：一个浏览器安全策略（保证浏览器能更加安全的方式运行）
    • 规定：协议名/域名/端口 三者必须完全一致

解决跨域的方法

  • jsonp
    • 优点：兼容性好
    • 缺点：只能发 GET 请求
  • cors
    • 优点：能处理任意方式请求
    • 缺点：兼容性稍差
  • 代理服务器
    • 正向代理服务器
      • 优点：开发时配置，比较简单（目标服务器不需要做任何操作）
      • 缺点：只能用于开发阶段，项目上线还是得用 cors/nginx 方案解决
    • 反向代理服务器（nginx）

跨域是谁负责检查

  • 浏览器
  • 结论
    • 服务器与服务器之间不存在跨域问题
    • 只有浏览器有跨域问题

代理服务器

请求方式

  • 原始方式：直接发送给目标服务器，浏览器跟目标服务器的协议名/域名/端口不同，产生跨域，导致请求没有结果
  • 代理服务器方式：
    • 请求先发给代理服务器，因为浏览器跟代理服务器之间符合同源策略，所以不会产生跨域
    • 代理服务器将请求转发到目标服务器上，服务器与服务器间不存在跨域问题

async 兼容性问题

  1. 问题：
    不支持 ES6 高级语法（如 async 函数），即使有了 babel，但是现在 Babel 配置只能编译一些简单的 ES6 语法。

    • 解决：@babel/polyfill：专门做 ES6 以上高级语法兼容性处理
    • 下载 npm i @babel/polyfill
    • 引入使用 import '@babel/polyfill'

  2. 新问题：
    我们现在只需要做 async 兼容性处理，但是一旦引用@babel/polyfill，就做所有的兼容性处理，文件变大。

    • 解决：core-js：来做按需加载（按照实际需求加载指定文件），源代码使用哪些有兼容性问题代码，自动引入需要做兼容性问题的文件，其他的不需要引入
    • 下载 npm i core-js
    • 更改 webpack 中的 use 下的 options 下的 presets

vueX

vueX 的基本介绍

  • 一个用于集中式管理多个组件共享状态数据的工具
  • 这个状态自管理应用包含以下几个部分：
    • state，驱动应用的数据源；
    • view，以声明方式将 state 映射到视图；
    • actions，响应在 view 上的用户输入导致的状态变化。
