vue-router 的介绍

vue-router 的理解

  1. vue 的一个插件库
  2. 专门用来实现一个 SPA 应用
  3. 基于 vue 的项目基本都会用到此库
  4. 中文文档: http://router.vuejs.org/zh-cn/
  5. 下载: npm install vue-router -S

SPA 的理解

  1. 单页 Web 应用（single page web application，SPA）
  2. 整个应用只有一个完整的页面
  3. 点击页面中的链接不会刷新页面, 本身也不会向服务器发请求
  4. 当点击路由链接时, 只会做页面的局部更新
  5. 数据都需要通过 ajax 请求获取, 并在前端异步展现

路由的理解

  1. 什么是路由?

    • 一个路由就是一个映射(对应)关系(key:value)
    • key 为路由路径 path, value 可能是 function/component

  2. 路由分类

    • 后台路由: node 服务器端路由, value 是 function, 用来处理客户端提交的请求并返回一个响应数据
    • 前台路由: 浏览器端路由, value 是 component, 当请求的是路由 path 时, 浏览器端没有发送 http 请求, 但界面会更新显示对应的组件

  3. 后台路由

    • 注册路由:
      • app.get(path, function(req, res))
      • router.get(path, function(req, res))
    • 当 node 接收到一个请求时, 根据请求路径找到匹配的路由, 调用路由中的函数来处理请求, 返回响应数据

  4. 前台路由

    • 注册路由
    • 当浏览器的 path 变为/about 时, 当前路由组件就会变为 About 组件

vue-router 的两个全局组件

vueRouter 除了给实例对象添加属性，还添加了两个全局组件

  1. <router-link class="list-group-item" to="/about"></router-link> 路由连接

    • 相当于 a 标签
    • 特点：不会发送请求，不会刷新页面，只会更新浏览器地址
    • 点击功能;
      1. 改变浏览器地址
      2. 给当前 a 标签添加一些类名
        .router-link-exact-active
        .router-link-active

  2. <router-view></router-view>

  • 显示当前路由组件
  • 当前路由组件是指那个？
    路由配置中 的 path 是否匹配上当前浏览器地址，匹配上了，component 就是当前组件

路由组件的生命周期

  • 当你从 a 组件切换成 b 组件 b 组件会创建 a 组件会卸载
  • 当你从 b 组件切换到 a 组件 a 组件会创建 b 组件会卸载
  • 当你从 c 组件切换到 c 组件 c 组件即不会创建也不会卸载，而是复用之前的组件（仅仅更新组件）

由于复用组件不会更新，因此不会重复调用 created（）中的函数，

  • 利用监视属性解决问题

两种路由模式

  1. hash 默认
    • 优点：兼容性极好
    • 缺点：请求路由带#
  2. history
    • 优点：请求路由不带#
    • 缺点：兼容性稍微差点（只支持 ie10 上）
  3. 总结：开发中一般使用 history 模式
  4. 问题
    • hash 模式：
      • hash 模式刷新浏览器时，向服务器发送请求，请求地址 http：//localhost:9527/
      • 服务器返回相应的响应，响应中包含 js 代码，js 代码解析时就会根据请求路径/#/home/news 加载指定的路由组件进行显示
    • history 模式
      • history 模式刷新浏览器时，向服务器发送请求，请求地址 http：//localhost:9527/home/news
      • 服务器并不能处理/home/news 只能处理/，所以找不到相应的路由处理 返回 404
  5. 解决
    • 开发的解决方案：一旦访问开发服务器 404，就返回 index。html 给你（相当于请求所有的开发服务器地址，都相当于请求 http：//localhost:9527/）
    • 通过配置 webpack 来解决：historyApiFallback：true，
    • 此时产生了新的问题：built.js 和 bootstrap.css 访问路径出了问题，根本原因是：输出路径是相对路径
      • 比如：当前路径是：http：//localhost：9527/home/news
      • 相对路径会以 http：//localhost：9527/home （去掉最后一层路径）
    • 解决将输出路径改成/开头，路径一定会以服务器根路径补充，实际路径 http：//localhost：9527/
    • 最终：
      • index.html 需要将 href=”./bootstrap.css”改成 href=”/bootstrap.css“
      • webpack.config.js 需要加上 output 上增加 publicPath：’/‘
